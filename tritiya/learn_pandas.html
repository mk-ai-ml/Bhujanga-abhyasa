📊 Series & DataFrame

Series and DataFrame are the fundamental data structures in Pandas. A Series is a 1D array with labels, while a DataFrame is a 2D table with rows and columns. Both are built on NumPy arrays but add powerful indexing and alignment capabilities.

series_dataframe_basics.py
import pandas as pd

# Create a Series from a list
s = pd.Series([10, 20, 30], index=['a', 'b', 'c'])
print(s)

# Create a Series from a dictionary
s_dict = pd.Series({'name': 'Alice', 'age': 25})

# Create a DataFrame from a dictionary of lists
df = pd.DataFrame({
    'name': ['Alice', 'Bob', 'Charlie'],
    'age': [25, 30, 35],
    'city': ['NYC', 'LA', 'Chicago']
})

# Check data types and shape
print(df.dtypes)      # Column data types
print(df.shape)       # (3, 3) - 3 rows, 3 columns
print(df.info())       # Detailed information
Series vs DataFrame

Think of a Series as a column in Excel, and a DataFrame as the entire spreadsheet. Both have an index (row labels) and use label-based or position-based access. DataFrames also have column names.

🎯 Selecting Data

Pandas provides multiple ways to select data: loc uses labels, iloc uses integer positions, at gets a single value, and boolean indexing filters rows. Choosing the right method is crucial for clean, efficient code.

selecting_data.py
df = pd.DataFrame({
    'product': ['A', 'B', 'C'],
    'price': [100, 200, 150],
    'quantity': [5, 3, 8]
})

# loc: label-based selection
print(df.loc[0])          # Get row with index label 0
print(df.loc[0, 'price'])  # Get specific cell: 100

# iloc: position-based selection
print(df.iloc[0])          # First row
print(df.iloc[0, 1])       # First row, second column

# Boolean filtering
expensive = df[df['price'] > 100]
print(expensive)  # Rows where price > 100

# Using query() for cleaner filtering
result = df.query('price > 100 and quantity >= 3')
print(result)
loc vs iloc Gotcha

loc uses labels (index names) and is inclusive on both ends: df.loc[0:2] includes rows 0, 1, and 2. iloc uses integer positions and is exclusive on the right: df.iloc[0:2] gets rows 0 and 1 only.

✏️ Adding & Modifying

Add new columns with direct assignment, modify existing ones with assign(), insert at specific positions, or drop unwanted columns. These operations are essential for data preparation.

adding_modifying.py
df = pd.DataFrame({
    'name': ['Alice', 'Bob'],
    'salary': [50000, 60000]
})

# Add a new column directly
df['bonus'] = df['salary'] * 0.1

# Use assign() for chaining (non-mutating)
df2 = df.assign(department=['Engineering', 'Sales'])

# Insert at specific position
df.insert(1, 'id', [101, 102])

# Drop columns
df_clean = df.drop(columns=['bonus'])

# Rename columns
df = df.rename(columns={'name': 'employee_name'})
Mutating vs Non-Mutating

Direct assignment and drop() mutate the original DataFrame. Use assign() to create a new DataFrame without modifying the original. Pass inplace=True to make mutations explicit.

📈 Sorting & Ranking

Sort rows by one or more columns, sort by index, rank values, or extract the top N rows. These operations help you organize and analyze data hierarchically.

sorting_ranking.py
df = pd.DataFrame({
    'name': ['Alice', 'Bob', 'Charlie'],
    'score': [85, 92, 78],
    'age': [25, 30, 28]
})

# Sort by one column (ascending)
sorted_df = df.sort_values('score')

# Sort by multiple columns (descending)
sorted_df = df.sort_values(['score', 'age'], ascending=False)

# Sort by index
sorted_df = df.sort_index()

# Rank values (1=lowest, ties get average rank)
df['rank'] = df['score'].rank()

# Get top N rows
top_2 = df.nlargest(2, 'score')
bottom_1 = df.nsmallest(1, 'score')

❓ Missing Data

Real data is messy. Detect missing values with isna(), drop them with dropna(), fill them with fillna(), or interpolate between values. Handling missing data is critical for accurate analysis.

missing_data.py
df = pd.DataFrame({
    'A': [1, 2, None, 4],
    'B': [5, None, 7, 8]
})

# Detect missing values
print(df.isna())        # Boolean mask
print(df.isnull())      # Same as isna()

# Drop rows with any missing value
df_clean = df.dropna()

# Fill missing values with a constant
df_filled = df.fillna(0)

# Forward fill (propagate last valid value)
df_ffill = df.fillna(method='ffill')

# Interpolate (linear by default)
df_interp = df.interpolate()
Be Careful with dropna()

Dropping rows with missing values can lose important data. Consider how much data you'll lose: df.dropna().shape vs df.shape. Sometimes filling or interpolating is better.

👥 GroupBy

GroupBy is the split-apply-combine pattern: partition data by a column, apply an operation to each group, and combine results. It's powerful for aggregations, transformations, and filtering.

groupby_examples.py
df = pd.DataFrame({
    'department': ['Sales', 'Sales', 'IT', 'IT'],
    'salary': [50000, 60000, 70000, 80000]
})

# Simple aggregation
avg_salary = df.groupby('department')['salary'].mean()

# Multiple aggregations
stats = df.groupby('department').agg({
    'salary': ['mean', 'sum', 'count']
})

# Transform: return result same shape as input
df['dept_avg'] = df.groupby('department')['salary'].transform('mean')

# Filter: keep groups meeting a condition
large_depts = df.groupby('department').filter(lambda x: len(x) > 2)

🔗 Merging & Joining

Combine DataFrames using merge() (SQL-like joins), concat() (row/column stacking), or join() (by index). Understanding join types is essential for combining datasets.

merging_joining.py
df1 = pd.DataFrame({'id': [1, 2], 'name': ['A', 'B']})
df2 = pd.DataFrame({'id': [1, 2], 'score': [90, 85]})

# Inner join: only matching rows
inner = df1.merge(df2, on='id', how='inner')

# Left join: keep all from left, match from right
left = df1.merge(df2, on='id', how='left')

# Outer join: keep all from both
outer = df1.merge(df2, on='id', how='outer')

# Concatenate rows
stacked = pd.concat([df1, df2], axis=0)

# Join by index
joined = df1.join(df2, lsuffix='_left', rsuffix='_right')
Join Type Left Rows Right Rows Use Case
inner Matching only Matching only Find common records
left All Matching Enrich left data
right Matching All Enrich right data
outer All All Complete picture

🔄 Reshaping

Transform data layout with pivot() (wide format), pivot_table() (aggregation pivot), melt() (long format), and stack()/unstack(). Reshaping is fundamental to exploratory analysis.

reshaping.py
df = pd.DataFrame({
    'date': ['2024-01', '2024-01', '2024-02', '2024-02'],
    'product': ['A', 'B', 'A', 'B'],
    'sales': [100, 200, 150, 250]
})

# Pivot: create columns from values (no aggregation)
pivot = df.pivot(index='date', columns='product', values='sales')

# Pivot table: aggregate while pivoting
pivot_agg = df.pivot_table(index='date', columns='product',
                               values='sales', aggfunc='sum')

# Melt: transform wide to long format
melted = df.melt(id_vars=['date'],
                  value_vars=['A', 'B'])

# Stack: pivot level of columns to index
stacked = pivot.stack()

# Unstack: pivot level of index to columns
unstacked = stacked.unstack()

⏰ String & DateTime

Process text with the .str accessor and dates/times with the .dt accessor. Convert to datetime with to_datetime() and resample time series with resample().

string_datetime.py
df = pd.DataFrame({
    'email': ['alice@example.com', 'bob@example.com'],
    'date': ['2024-01-15', '2024-02-20']
})

# String operations with .str accessor
df['name'] = df['email'].str.split('@').str[0]
df['domain'] = df['email'].str.extract(r'@(.*)')

# Convert to datetime
df['date'] = pd.to_datetime(df['date'])

# DateTime operations with .dt accessor
df['year'] = df['date'].dt.year
df['month'] = df['date'].dt.month
df['day_name'] = df['date'].dt.day_name

# Set date as index and resample
df_ts = df.set_index('date')
weekly = df_ts.resample('W').sum()

🪟 Window Functions

Apply operations over rolling/expanding windows with rolling(), expanding(), and ewm(). Also shift values and compute differences with shift() and diff().

window_functions.py
df = pd.DataFrame({
    'price': [100, 102, 98, 105, 110]
})

# Rolling window (moving average)
df['ma_3'] = df['price'].rolling(window=3).mean()

# Expanding window (cumulative)
df['cumsum'] = df['price'].expanding().sum()

# Exponential weighted moving average
df['ewma'] = df['price'].ewm(span=3).mean()

# Shift values (lag/lead)
df['previous'] = df['price'].shift(1)
df['next'] = df['price'].shift(-1)

# Percent change
df['pct_change'] = df['price'].pct_change()

🎨 Advanced Patterns

Use pipe() for method chaining, MultiIndex for hierarchical data, categorical dtype for memory efficiency, and apply() for custom operations.

advanced_patterns.py
df = pd.DataFrame({
    'category': ['A', 'B', 'A'],
    'value': [10, 20, 30]
})

# Method chaining with pipe()
result = (df
  .pipe(lambda x: x[x['value'] > 10])
  .groupby('category')
  .agg('sum')
)

# Categorical dtype for memory efficiency
df['category'] = df['category'].astype('category')

# Apply custom function
df['squared'] = df['value'].apply(lambda x: x ** 2)

# MultiIndex for hierarchical data
index = pd.MultiIndex.from_tuples(
    [('A', 'x'), ('A', 'y'), ('B', 'x')],
    names=['group', 'sub']
)
df_multi = pd.DataFrame({'val': [1, 2, 3]}, index=index)
pipe() for Chaining

Use pipe() to pass a DataFrame through custom functions, enabling readable multi-step transformations without intermediate variables. This makes code more declarative and testable.

⚡ Performance

Optimize Pandas for large datasets: use vectorized operations, convert to categorical dtype, use query() and eval(), avoid row-by-row iteration, and profile memory usage.

performance_tips.py
df = pd.DataFrame({
    'A': range(1000000),
    'B': range(1000000),
    'category': ['X', 'Y'] * 500000
})

# Slow: row-by-row iteration
# for idx, row in df.iterrows():  # AVOID
#     df.loc[idx, 'C'] = row['A'] + row['B']

# Fast: vectorized operation
df['C'] = df['A'] + df['B']

# Use categorical for low cardinality columns
df['category'] = df['category'].astype('category')

# Use query() for complex filters
result = df.query('A > 500 and B < 1000')

# Check memory usage
print(df.memory_usage(deep=True).sum() / 1024 ** 2)  # MB

# Use dtypes to reduce memory
df['A'] = df['A'].astype('int32')  # instead of int64
Vectorization is Key

Pandas is built on NumPy's vectorized operations. Using iterrows(), apply() with lambda, or loops can be 100x slower. Always look for vectorized alternatives first.
