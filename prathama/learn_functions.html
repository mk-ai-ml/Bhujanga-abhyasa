Python Functions

Functions are the fundamental building blocks of Python programming. They allow you to encapsulate logic, reuse code, and create abstractions that make your programs cleaner, more maintainable, and infinitely more powerful. From simple utilities to complex generators and decorators, mastering functions is essential to writing Pythonic code.

1. Defining Functions

A function is a reusable block of code that performs a specific task. In Python, you define functions using the def keyword, followed by the function name, parameters, and a colon.

Basic Function Definition

functions_basics.py
def greet(name):
    """Return a personalized greeting."""
    return f"Hello, {name}!"

result = greet("Alice")
print(result)  # Output: Hello, Alice!

Docstrings & Documentation

Docstrings (triple-quoted strings immediately after the def line) document what your function does. They're essential for code clarity and are accessible via the help() function.

docstrings.py
def calculate_area(radius):
    """Calculate the area of a circle.

    Args:
        radius (float): The radius of the circle.

    Returns:
        float: The area of the circle.
    """
    import math
    return math.pi * radius ** 2

print(calculate_area(5))  # Output: 78.53981633974483
Pro Tip: Function Naming

Use lowercase with underscores (snake_case) for function names. Make names descriptive: calculate_total_price() is clearer than calc().

Return Statement

The return keyword sends a value back to the caller. Without a return, a function returns None.

return_demo.py
def add(a, b):
    return a + b

def print_message(msg):
    print(msg)  # No return statement

result1 = add(3, 5)          # result1 = 8
result2 = print_message("Hi")  # result2 = None

2. Parameters & Arguments

Parameters are the variables in the function definition; arguments are the values you pass when calling the function. Python supports several types: positional, keyword, and default values.

Positional Arguments

Arguments matched by position, in order. The caller must pass them in the correct order.

positional.py
def subtract(a, b):
    return a - b

subtract(10, 3)  # = 7 (order matters!)
subtract(3, 10)  # = -7 (different result)

Keyword Arguments

Arguments specified by name, allowing any order. Improves readability for functions with many parameters.

keyword.py
def describe_person(name, age, city):
    return f"{name} is {age} years old and lives in {city}"

# All of these are equivalent:
describe_person("Alice", 30, "NYC")
describe_person(age=30, name="Alice", city="NYC")
describe_person("Alice", city="NYC", age=30)

Default Values

Parameters can have default values. If the caller doesn't provide an argument, the default is used.

defaults.py
def power(base, exponent=2):
    return base ** exponent

power(5)        # = 25 (uses default: 5^2)
power(5, 3)     # = 125 (5^3)
power(exponent=4, base=2)  # = 16 (2^4)
Caution: Mutable Defaults

Never use mutable objects (lists, dicts) as defaults. They persist across calls, leading to unexpected behavior. Use None instead and create new instances inside the function.

mutable_default.py
# ❌ WRONG:
def add_item_wrong(item, items=[]):
    items.append(item)
    return items

# ✅ CORRECT:
def add_item_right(item, items=None):
    if items is None:
        items = []
    items.append(item)
    return items

3. *args and **kwargs

These allow functions to accept a variable number of arguments. *args captures positional arguments as a tuple; **kwargs captures keyword arguments as a dictionary.

*args: Variable Positional Arguments

The asterisk unpacks multiple positional arguments into a tuple named args (the name can vary, but the asterisk is key).

args_demo.py
def sum_all(*args):
    return sum(args)

sum_all(1)              # = 1, args = (1,)
sum_all(1, 2, 3)        # = 6, args = (1, 2, 3)
sum_all(10, 20, 30, 40)  # = 100

**kwargs: Variable Keyword Arguments

Double asterisks collect keyword arguments into a dictionary named kwargs.

kwargs_demo.py
def print_info(**kwargs):
    for key, value in kwargs.items():
        print(f"{key}: {value}")

print_info(name="Alice", age=30, city="NYC")
# Output:
# name: Alice
# age: 30
# city: NYC

Combining All Three

You can mix positional, *args, keyword, and **kwargs. The order must be: positional → *args → keyword-only → **kwargs.

combined.py
def flexible_func(a, b, *args, debug=False, **kwargs):
    print(f"a={a}, b={b}")
    print(f"args={args}")
    print(f"debug={debug}")
    print(f"kwargs={kwargs}")

flexible_func(1, 2, 3, 4, debug=True, x=10, y=20)

Unpacking with * and **

You can also use * and ** when calling functions to unpack sequences and dictionaries.

unpacking.py
def multiply(a, b, c):
    return a * b * c

# Unpacking a list with *
numbers = [2, 3, 4]
result = multiply(*numbers)  # multiply(2, 3, 4)

# Unpacking a dict with **
params = {'a': 2, 'b': 3, 'c': 4}
result = multiply(**params)

4. Return Values

Functions return data using the return statement. You can return any Python object: numbers, strings, lists, dicts, even other functions.

Single vs. Multiple Returns

return_values.py
# Single return value
def square(x):
    return x ** 2

# Multiple returns (as tuple)
def divmod_custom(a, b):
    return a // b, a % b

quotient, remainder = divmod_custom(17, 5)
# quotient = 3, remainder = 2

Early Return

Use return early to exit a function and avoid deep nesting. This improves readability and reduces cognitive load.

early_return.py
# ❌ Nested (hard to read)
def process_bad(value):
    if value > 0:
        if value < 100:
            result = value * 2
        else:
            result = -1
    else:
        result = -1
    return result

# ✅ Early return (clear flow)
def process_good(value):
    if value <= 0 or value >= 100:
        return -1
    return value * 2

None as Default Return

Functions without an explicit return statement return None. Use this for functions that perform side effects (printing, updating state) rather than returning values.

none_return.py
def log_message(msg):
    print(f"[LOG] {msg}")
    # Implicitly returns None

result = log_message("Hello")
print(result)  # Output: None

5. Scope & Variables

Scope determines where a variable is accessible. Python uses the LEGB rule: Local → Enclosing → Global → Built-in.

Python Scope Hierarchy (LEGB)
Local: Inside a function
Enclosing: In an outer function (nested)
Global: At module level
Built-in: Python's built-in namespace

Local Scope

Variables defined inside a function exist only within that function. They're created when the function is called and destroyed when it exits.

local_scope.py
def my_func():
    x = 10  # Local variable
    return x

my_func()   # = 10
# print(x)  # ❌ NameError: x not defined

Global Scope

Variables defined outside functions are global. Functions can read global variables, but to modify them, use the global keyword.

global_scope.py
x = 5  # Global

def read_global():
    print(x)  # Can read x

def modify_global():
    global x
    x = 10  # Now modifies global x

read_global()      # Output: 5
modify_global()
read_global()      # Output: 10

Enclosing Scope & nonlocal

In nested functions, the enclosing scope refers to variables in an outer function. Use nonlocal to modify them.

enclosing_scope.py
def outer():
    x = 5  # Enclosing scope

    def inner():
        nonlocal x
        x = 10  # Modifies outer's x

    inner()
    return x  # = 10

outer()  # = 10
Best Practice: Limit Global Use

Overuse of global can make code hard to debug. Prefer passing variables as parameters and returning values.

6. Recursion

Recursion is when a function calls itself. Every recursive function must have a base case (where it stops) and a recursive case (where it calls itself with simpler input).

Anatomy of Recursion

factorial.py
def factorial(n):
    # Base case: stop recursion
    if n <= 1:
        return 1

    # Recursive case: call itself with n-1
    return n * factorial(n - 1)

factorial(5)  # = 120

Call Stack Visualization

When factorial(5) is called, each recursive call stacks on top of the previous one until the base case is reached, then results "bubble up."

factorial(1) → 1
↑
2 × result = 2
↑
3 × result = 6
↑
4 × result = 24
↑
5 × result = 120

Classic Examples: Fibonacci

fibonacci.py
# Simple but slow (lots of redundant calls)
def fib_slow(n):
    if n <= 1:
        return n
    return fib_slow(n - 1) + fib_slow(n - 2)

# Optimized with memoization
def fib_fast(n, memo={}):
    if n in memo:
        return memo[n]
    if n <= 1:
        return n

    memo[n] = fib_fast(n - 1, memo) + fib_fast(n - 2, memo)
    return memo[n]

Recursion Depth Limit

Python has a default recursion depth limit (usually 1000) to prevent stack overflow. Use sys.setrecursionlimit() carefully, or prefer iteration for deep recursions.

recursion_limit.py
import sys

# Check current limit
print(sys.getrecursionlimit())  # Usually 1000

# Increase if needed (use cautiously)
sys.setrecursionlimit(5000)

# Better: use iteration instead for deep recursions
def sum_iterative(n):
    total = 0
    for i in range(n + 1):
        total += i
    return total
When to Use Recursion

Use recursion for problems with recursive structure (trees, graphs, divide-and-conquer). Use iteration for simple loops. Remember: "If it can be done iteratively, it should be done iteratively."

7. Lambda Functions

Lambda functions are small, anonymous functions defined with the lambda keyword. They're useful for short operations but shouldn't replace regular functions for complex logic.

Basic Lambda Syntax

lambda_basics.py
# Lambda: lambda arguments: expression

# Simple square function
square = lambda x: x ** 2
square(5)  # = 25

# Multiple arguments
add = lambda a, b: a + b
add(3, 7)  # = 10

# With default argument
power = lambda x, p=2: x ** p
power(3)      # = 9
power(3, 3)   # = 27

When to Use Lambdas

Use lambdas for short, simple operations, especially as arguments to higher-order functions like map(), filter(), and sorted().

lambda_usage.py
# Sort tuples by second element
students = [("Alice", 85), ("Bob", 92), ("Charlie", 78)]
sorted_students = sorted(students, key=lambda x: x[1])

# Double each number in a list
numbers = [1, 2, 3, 4, 5]
doubled = list(map(lambda x: x * 2, numbers))
# Result: [2, 4, 6, 8, 10]

# Filter even numbers
evens = list(filter(lambda x: x % 2 == 0, numbers))
# Result: [2, 4]
Lambda Limitations

Lambdas can only contain a single expression. For multi-line logic, use a regular def function. Also, lambdas can't have docstrings, so they're harder to document.

8. Higher-Order Functions

Higher-order functions take other functions as arguments or return functions. They're powerful tools for functional programming and code abstraction.

Functions as Arguments

higher_order.py
def apply_twice(func, x):
    return func(func(x))

def increment(n):
    return n + 1

result = apply_twice(increment, 5)
# increment(increment(5)) = increment(6) = 7

Built-in Higher-Order: map(), filter(), reduce()

map_filter_reduce.py
from functools import reduce

numbers = [1, 2, 3, 4, 5]

# map: apply function to each element
squared = list(map(lambda x: x ** 2, numbers))
# [1, 4, 9, 16, 25]

# filter: keep elements where condition is True
evens = list(filter(lambda x: x % 2 == 0, numbers))
# [2, 4]

# reduce: combine all elements into one value
product = reduce(lambda a, b: a * b, numbers)
# 1 * 2 * 3 * 4 * 5 = 120

Returning Functions

Functions can return other functions, enabling powerful patterns like function factories.

function_factory.py
def make_multiplier(n):
    def multiplier(x):
        return x * n
    return multiplier

# Create specialized functions
double = make_multiplier(2)
triple = make_multiplier(3)

double(5)  # = 10
triple(5)  # = 15

9. Closures

A closure is a function that "remembers" variables from its enclosing scope, even after the outer function has returned. Closures are fundamental to decorators, callbacks, and functional programming.

Anatomy of a Closure

closure_basic.py
def outer(x):
    # Inner function "closes over" x
    def inner(y):
        return x + y
    return inner

# add_5 is a closure with x=5 captured
add_5 = outer(5)
add_5(3)  # = 8

add_10 = outer(10)
add_10(3)  # = 13

Practical: make_counter()

A classic closure pattern: create a counter function that maintains state.

counter.py
def make_counter(start=0):
    count = start

    def increment():
        nonlocal count
        count += 1
        return count

    return increment

# Each call to make_counter creates a new closure
counter1 = make_counter()
counter2 = make_counter(100)

counter1()  # = 1
counter1()  # = 2
counter1()  # = 3

counter2()  # = 101
counter2()  # = 102
Closures in Real Code

Closures enable decorators (wrapping functions), event handlers (callbacks), and data encapsulation without classes. They're especially useful in event-driven and functional programming.

10. Decorators

Decorators wrap a function to modify its behavior without changing its code. They use closures and higher-order functions to add functionality like logging, timing, caching, and retry logic.

Basic Decorator Pattern

decorator_basic.py
def my_decorator(func):
    def wrapper(*args, **kwargs):
        print(f"Calling {func.__name__}")
        result = func(*args, **kwargs)
        print(f"Finished {func.__name__}")
        return result
    return wrapper

# Using the decorator with @ syntax
@my_decorator
def say_hello(name):
    print(f"Hello, {name}!")

say_hello("Alice")
# Output:
# Calling say_hello
# Hello, Alice!
# Finished say_hello

Practical: @timing Decorator

Measure how long a function takes to execute.

timing_decorator.py
import time
from functools import wraps

def timing(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        elapsed = time.time() - start
        print(f"{func.__name__} took {elapsed:.4f}s")
        return result
    return wrapper

@timing
def slow_function():
    time.sleep(2)
    return "Done"

slow_function()  # slow_function took 2.0005s

Practical: @memoize Decorator

Cache function results to avoid redundant computation.

memoize_decorator.py
from functools import lru_cache

# Built-in memoization (Python 3.2+)
@lru_cache(maxsize=128)
def fibonacci(n):
    if n <= 1:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)

# Much faster than naive recursion!
fibonacci(35)  # Instant (with memoization)

Decorators with Arguments

To pass arguments to a decorator, add another layer of nesting.

decorator_args.py
def repeat(times):
    # Decorator factory
    def decorator(func):
        def wrapper(*args, **kwargs):
            results = []
            for _ in range(times):
                results.append(func(*args, **kwargs))
            return results
        return wrapper
    return decorator

@repeat(times=3)
def greet(name):
    return f"Hello, {name}!"

greet("Bob")
# ['Hello, Bob!', 'Hello, Bob!', 'Hello, Bob!']
Use functools.wraps()

Always use @functools.wraps(func) in your decorator's wrapper to preserve the original function's metadata (name, docstring, etc.).

11. Generators

Generators are functions that use yield instead of return. They produce values one at a time, using less memory than creating entire lists. Perfect for large datasets or infinite sequences.

yield: Lazy Evaluation

generator_basic.py
# Regular function: returns entire list
def range_list(n):
    result = []
    for i in range(n):
        result.append(i)
    return result

# Generator: yields one value at a time
def range_gen(n):
    for i in range(n):
        yield i

# range_list(5) creates [0,1,2,3,4] in memory
# range_gen(5) yields on demand; memory efficient

Iterating Over Generators

generator_iteration.py
def fibonacci_gen():
    a, b = 0, 1
    while True:
        yield a
        a, b = b, a + b

# Use in a for loop
for fib in fibonacci_gen():
    if fib > 100:
        break
    print(fib)

# Or manually with next()
gen = fibonacci_gen()
next(gen)  # = 0
next(gen)  # = 1
next(gen)  # = 1
next(gen)  # = 2

Generator Expressions

Similar to list comprehensions, but with parentheses instead of brackets. Generate values lazily.

generator_expressions.py
# List comprehension (eager; all values at once)
squares_list = [x ** 2 for x in range(5)]

# Generator expression (lazy; values on demand)
squares_gen = (x ** 2 for x in range(5))

sum(squares_list)  # Works fine
sum(squares_gen)   # Also works, more memory efficient
When to Use Generators

Use generators for: large datasets, streaming data, infinite sequences, and when you only need values one at a time. They're memory-efficient and can improve performance significantly.

12. Functional Composition

Composition combines simple functions into more complex ones. It's about building complex operations from smaller, reusable pieces.

Simple Function Composition

composition.py
# Small, single-purpose functions
def add_one(x):
    return x + 1

def double(x):
    return x * 2

def square(x):
    return x ** 2

# Compose manually
x = 3
result = square(double(add_one(x)))
# (3+1)*2 squared = 4*2 = 8, 8^2 = 64

Compose Function

Create a higher-order function that composes functions together.

compose_function.py
from functools import reduce

def compose(*funcs):
    """Compose functions right-to-left: compose(f,g,h)(x) = f(g(h(x)))
    """
    def composed(x):
        return reduce(lambda val, func: func(val),
                         reversed(funcs), x)
    return composed

# Now we can compose cleanly
pipeline = compose(square, double, add_one)
pipeline(3)  # = 64

Pipe: Left-to-Right Composition

Sometimes it's more readable to apply functions left-to-right, like a pipeline.

pipe.py
from functools import reduce

def pipe(*funcs):
    """Pipe functions left-to-right: pipe(f,g,h)(x) = h(g(f(x)))
    """
    def piped(x):
        return reduce(lambda val, func: func(val), funcs, x)
    return piped

# Read top-to-bottom, like a pipeline
pipeline = pipe(add_one, double, square)
pipeline(3)  # = 64

Partial Application

Fix some arguments of a function to create a new, specialized function.

partial.py
from functools import partial

def multiply(a, b):
    return a * b

# Create specialized functions
double = partial(multiply, 2)  # fix a=2
triple = partial(multiply, 3)  # fix a=3

double(5)  # = 10
triple(5)  # = 15
Benefits of Composition

Composition promotes code reusability, testability, and clarity. Small, pure functions are easier to reason about and combine in different ways.
