🔤 Python Strings

Master the art of text manipulation, from creation to advanced algorithms. Strings are immutable sequences of characters that power everything from simple messages to complex natural language processing.

1 String Basics

Strings are one of Python's fundamental data types. A string is an immutable sequence of Unicode characters. Whether you're building a web application, processing logs, or analyzing text, understanding string fundamentals is essential.

Creating Strings

Python allows you to create strings using single quotes, double quotes, or triple quotes. All three are equivalent—the choice is yours, though consistency matters for readability.

string_creation.py
# Single quotes
greeting = 'Hello, World!'

# Double quotes
message = "Python is awesome"

# Triple quotes (allow newlines)
poem = """Roses are red,
Violets are blue,
Python's elegant,
And Strings are too"""

# Empty string
empty = ""

# String with special characters
special = "Line 1\nLine 2\tTabbed"
String Quotes Equivalence

These three declarations are identical:

s1 = 'text'
s2 = "text"
s3 = """text"""
String Immutability

Strings in Python are immutable. Once created, you cannot change individual characters. Operations that appear to modify a string actually create a new string.

immutability.py
text = "Python"

# This will raise a TypeError
# text[0] = 'J'  # TypeError: 'str' object does not support item assignment

# Instead, create a new string
new_text = "J" + text[1:]
print(new_text)  # Output: Jython

# The original remains unchanged
print(text)  # Output: Python
Why Immutability? Immutability makes strings safe to use as dictionary keys and in sets. It also enables Python to optimize string storage and sharing behind the scenes.

Negative Indexing

Negative indices count backward from the end of the string. -1 refers to the last character, -2 to the second-to-last, and so on.

text = "Python"

# Negative indexing
print(text[-1])  # Output: n (last character)
print(text[-2])  # Output: o
print(text[-6])  # Output: P (first character)

# Both approaches are equivalent
print(text[0] == text[-6])  # Output: True
Index Out of Range: Accessing an index that doesn't exist raises an IndexError. Always ensure your index is within valid bounds.

3 Slicing

Slicing extracts a substring from a string using the syntax string[start:stop:step]. This is one of Python's most powerful features for string manipulation.

Basic Slicing

Extract a contiguous subsequence using string[start:stop]. The start index is inclusive; the stop index is exclusive.

basic_slicing.py
text = "Python"

print(text[0:3])  # Output: Pyt (indices 0, 1, 2)
print(text[2:5])  # Output: tho (indices 2, 3, 4)
print(text[1:])   # Output: ython (from index 1 to end)
print(text[:4])   # Output: Pyth (from start to index 3)
print(text[:])    # Output: Python (entire string)

Step Parameter

The step parameter determines how many characters to skip. A negative step reverses the string.

slicing_step.py
text = "Python"

# Every 2nd character
print(text[0:6:2])  # Output: Pto

# Reverse the string
print(text[::-1])     # Output: nohtyP

# Every 3rd character, reversed
print(text[::-3])    # Output: nP
Reversing Strings: To reverse a string, use string[::-1]. This is a common idiom in Python and very efficient.
Negative Indices in Slices

You can use negative indices in slices just as you would with positive ones.

negative_slicing.py
text = "Python"

# Last 3 characters
print(text[-3:])  # Output: hon

# All except first and last
print(text[1:-1])  # Output: ytho

# Characters from -4 to -1 (exclusive)
print(text[-4:-1])  # Output: tho
4 Common String Methods

Python strings come with numerous built-in methods that perform common operations. These methods return new strings (since strings are immutable) and never modify the original.

Find substrings and replace occurrences with ease.

5 String Formatting

Python offers multiple ways to insert variables into strings. Modern Python (3.6+) favors f-strings, but all methods remain useful.

F-Strings (Python 3.6+)

F-strings are the most readable and performant method. Prefix a string with f and place expressions in curly braces.

fstring.py
name = "Alice"
age = 28
score = 95.678

# Simple variable insertion
print(f"Name: {name}, Age: {age}")

# Expressions in f-strings
print(f"Next year I'll be {age + 1}")

# Format specifiers
print(f"Score: {score:.2f}%")  # Output: Score: 95.68%

# Alignment and padding
print(f"{name:>10}")  # Right-aligned, width 10
print(f"{name:<10}")  # Left-aligned, width 10
print(f"{name:^10}")  # Center-aligned, width 10
.format() Method

The format() method works with any Python version and offers similar functionality to f-strings.

format_method.py
name = "Bob"
score = 87.456

# Positional arguments
print("Hello, {}!".format(name))

# Multiple placeholders
print("{} scored {:.1f}".format(name, score))

# Named arguments
print("{name} has {count} items".format(name="Carol", count=5))
Percent Operator (%)

The older % operator is less preferred but still found in legacy code.

percent_format.py
name = "Dave"
count = 42

# String (%s) and integer (%d)
print("Hello, %s!" % name)
print("There are %d items" % count)

# Multiple values (use tuple)
print("%s has %d points" % (name, count))
Best Practice: Use f-strings for new code. They're faster, more readable, and support full expressions. Use .format() for compatibility with older Python versions.

Additional testing methods for common patterns.

8 String Comparison & Sorting

Strings are compared lexicographically using their Unicode values. This enables sorting and equality checks.

Lexicographic Comparison

Strings are compared character by character based on Unicode values.

string_comparison.py
# Equality comparison
print("apple" == "apple")  # Output: True
print("apple" == "Apple")  # Output: False (case-sensitive)

# Ordering comparisons
print("apple" < "banana")  # Output: True (a < b)
print("apple" < "apricot") # Output: True (appl < apri)
print("zebra" > "apple")   # Output: True

# Unicode values matter
print("A" < "a")  # Output: True (65 < 97 in Unicode)
Lexicographic Ordering Example
Comparison Result Reason
"apple" == "apple" True Identical strings
"apple" < "apricot" True 'p' (112) < 'r' (114) at position 2
"apple" < "apple2" True Shorter string comes first
"Apple" < "apple" True 'A' (65) < 'a' (97)
Case-Insensitive Comparison

Convert to the same case before comparison for case-insensitive matching.

case_insensitive.py
user_input = "HELLO"

# Method 1: Convert both to lowercase

    print("Match!")

# Method 2: Use casefold() for more aggressive case folding
if user_input.casefold() == expected.casefold():

9 Regular Expressions Basics

Regular expressions (regex) provide a powerful way to search, match, and manipulate strings using patterns. The re module is Python's gateway to regex functionality.

Basic Pattern Matching

Use re.search() and re.match() to find patterns in strings.

regex_basics.py
import re

text = "The phone number is 123-456-7890"

# re.search() - finds pattern anywhere in string
match = re.search(r"\d{3}-\d{3}-\d{4}", text)
if match:
    print(match.group())  # Output: 123-456-7890

# re.match() - matches at string beginning
text2 = "123 Main Street"
match2 = re.match(r"\d+", text2)
if match2:
    print(match2.group())  # Output: 123
Finding All Matches

Use re.findall() to extract all matches.

regex_findall.py
import re

text = "Emails: john@example.com, jane@test.org, bob@mail.net"

# Find all email addresses
emails = re.findall(r"\w+@\w+\.\w+", text)
print(emails)
# Output: ['john@example.com', 'jane@test.org', 'bob@mail.net']
Common Regex Patterns

Essential patterns for everyday string processing.

Common Regex Metacharacters
Pattern Meaning Example
\d Any digit (0-9) \d{3} matches "123"
\w Word character (alphanumeric + _) \w+ matches "hello_123"
\s Whitespace character \s+ matches spaces/tabs

10 Encoding Patterns

Transform strings using encoding techniques like Caesar cipher, ROT13, run-length encoding, and simple compression methods.

Caesar Cipher

Shift each character by a fixed number of positions in the alphabet.

caesar_cipher.py
def caesar_cipher(text, shift=3):
    result = ""
    for char in text:
        if char.isalpha():
            base = ord('A') if char.isupper() else ord('a')
            shifted = chr(base + (ord(char) - base + shift) % 26)
            result += shifted
        else:
            result += char
    return result

print(caesar_cipher("Hello", 3))  # Output: Khoor
ROT13

A simple substitution cipher that rotates characters by 13 positions.

rot13.py
import codecs

text = "Hello World"

# Using codecs module
encoded = codecs.encode(text, 'rot_13')
print(encoded)  # Output: Uryyb Jbeyq

# Decode (ROT13 applied twice returns original)
decoded = codecs.encode(encoded, 'rot_13')
print(decoded)  # Output: Hello World
Run-Length Encoding

Compress strings by encoding runs of identical characters.

run_length_encoding.py
def rle_encode(text):
    if not text:
        return ""
    result = ""
    count = 1
    for i in range(len(text) - 1):
        if text[i] == text[i + 1]:
            count += 1
        else:
            result += str(count) + text[i]
            count = 1
    result += str(count) + text[-1]
    return result

print(rle_encode("aabbcc"))  # Output: 2a2b2c

11 String Algorithms

Classic algorithms that operate on strings: palindrome detection, anagram checking, bracket matching, and tokenization.

Palindrome Check

Determine if a string reads the same forward and backward.

palindrome.py
def is_palindrome(text):
    # Remove non-alphanumeric and convert to lowercase
    cleaned = "".join(char.lower() for char in text if char.isalnum())
    return cleaned == cleaned[::-1]

print(is_palindrome("A man, a plan, a canal: Panama"))  # Output: True
print(is_palindrome("Hello"))  # Output: False
Anagram Detection

Check if two strings are anagrams (contain the same characters in different order).

anagram.py
def is_anagram(text1, text2):
    # Remove spaces and convert to lowercase
    clean1 = text1.replace(" ", "").lower()
    clean2 = text2.replace(" ", "").lower()
    return sorted(clean1) == sorted(clean2)

print(is_anagram("listen", "silent"))  # Output: True
print(is_anagram("hello", "world"))  # Output: False
Bracket Matching

Verify that brackets are properly balanced using a stack approach.

bracket_matching.py
def is_balanced(text):
    stack = []
    pairs = {')': '(', ']': '[', '}': '{'}
    for char in text:
        if char in '([{':
            stack.append(char)
        elif char in ')]}':
            if not stack or stack[-1] != pairs[char]:
                return False
            stack.pop()
    return len(stack) == 0

print(is_balanced("({[]})"))  # Output: True
print(is_balanced("({[}])"))  # Output: False

Longest Common Substring

Find the longest substring present in multiple strings.

lcs.py
def longest_common_substring(text1, text2):
    longest = ""
    for i in range(len(text1)):
        for j in range(i + 1, len(text1) + 1):
            substr = text1[i:j]
            if substr in text2 and len(substr) > len(longest):
                longest = substr
    return longest

result = longest_common_substring("algorithm", "altruism")
print(result)  # Output: alg
Word Frequency Analysis

Count occurrences of each word in a text.

word_frequency.py
from collections import Counter
import re

def word_frequency(text):
    words = re.findall(r"\w+", text.lower())
    return Counter(words)

text = "the quick brown fox jumps over the lazy dog"
freq = word_frequency(text)
print(freq.most_common(3))
# Output: [('the', 2), ('quick', 1), ('brown', 1)]

URL Slug Generation

Convert text into URL-safe slugs.

slug_generation.py
import re

def generate_slug(text):
    # Convert to lowercase
    slug = text.lower()
    # Replace spaces with hyphens
    slug = re.sub(r"\s+", "-", slug)
    # Remove non-alphanumeric characters (except hyphens)
    slug = re.sub(r"[^\w-]", "", slug)
    # Remove consecutive hyphens
    slug = re.sub(r"-+", "-", slug)
    return slug.strip("-")

print(generate_slug("Hello, World! Welcome to Python"))
# Output: hello-world-welcome-to-python
