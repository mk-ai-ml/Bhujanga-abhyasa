📋 List Comprehensions Basics

List comprehensions provide a concise and powerful way to create new lists by applying an expression to each element in an iterable. They replace verbose loops with elegant, readable syntax while being faster than traditional iteration.

list_comp_basics.py
# Traditional way - verbose
squares = []
for x in range(5):
    squares.append(x ** 2)

# List comprehension - clean and fast
squares = [x ** 2 for x in range(5)]
# Result: [0, 1, 4, 9, 16]

# Double each element
numbers = [1, 2, 3, 4]
doubled = [x * 2 for x in numbers]
# Result: [2, 4, 6, 8]

# Convert strings to integers
strings = ['1', '2', '3']
ints = [int(s) for s in strings]
# Result: [1, 2, 3]

The basic syntax is: [expression for item in iterable]. The expression is evaluated for each item, and the results are collected into a new list. This is much faster than appending to a list in a loop.

Speed Advantage

List comprehensions are typically 2-3x faster than equivalent for loops because they're optimized at the bytecode level and avoid repeated function calls.

🔍 Filtering with Comprehensions

Add an optional if clause to filter items based on a condition. Only items that pass the test are included in the result.

filter_examples.py
# Keep only even numbers
numbers = [1, 2, 3, 4, 5, 6]
evens = [x for x in numbers if x % 2 == 0]
# Result: [2, 4, 6]

# Filter words by length
words = ['cat', 'elephant', 'dog', 'butterfly']
long_words = [w for w in words if len(w) > 3]
# Result: ['elephant', 'butterfly']

# Filter and transform together
prices = [10, 25, 15, 30, 5]
discounted = [p * 0.9 for p in prices if p > 10]
# Result: [22.5, 27.0]

The syntax with filtering is: [expression for item in iterable if condition]. The expression is only evaluated for items where the condition is True.

Filter Logic

You can use complex conditions with and, or, and not. Only items matching the condition appear in the result list.

🔄 Transforming Data

Comprehensions excel at transforming data—applying functions, changing types, or extracting parts of objects. You can chain multiple operations for powerful pipelines.

transform_examples.py
# Extract field from list of dicts
users = [
    {'name': 'Alice', 'age': 25},
    {'name': 'Bob', 'age': 30}
]
names = [u['name'] for u in users]
# Result: ['Alice', 'Bob']

# Chain multiple transforms
data = ['hello', 'WORLD', 'Python']
processed = [s.lower().replace('o', '0') for s in data]
# Result: ['hell0', 'w0rld', 'pyth0n']

# Apply built-in functions
string_nums = ['123', '456', '789']
as_ints = [int(x) for x in string_nums]
# Result: [123, 456, 789]

# Custom function application
def celsius_to_fahrenheit(c):
    return c * 9/5 + 32

celsius = [0, 10, 20, 30]
fahrenheit = [celsius_to_fahrenheit(c) for c in celsius]
# Result: [32.0, 50.0, 68.0, 86.0]
Chaining Operations

You can chain method calls (like .lower().replace()) inside comprehensions for powerful data transformations without intermediate variables.

🔗 Nested Comprehensions

Nested comprehensions iterate over multiple sequences, creating multi-dimensional structures or flattening nested data. They replace nested for loops with elegant syntax.

nested_examples.py
# Create multiplication table
table = [x * y for x in range(1, 4) for y in range(1, 4)]
# Result: [1, 2, 3, 2, 4, 6, 3, 6, 9]

# Create coordinate pairs
pairs = [(x, y) for x in [1, 2] for y in ['a', 'b']]
# Result: [(1, 'a'), (1, 'b'), (2, 'a'), (2, 'b')]

# Flatten a matrix
matrix = [[1, 2], [3, 4], [5, 6]]
flattened = [item for row in matrix for item in row]
# Result: [1, 2, 3, 4, 5, 6]

# Nested with filtering
data = [[1, 2, 3], [4, 5], [6, 7, 8]]
evens = [x for row in data for x in row if x % 2 == 0]
# Result: [2, 4, 6, 8]

Nested comprehensions read left-to-right like nested loops: [expr for outer in seq1 for inner in seq2]. The leftmost for is the outermost loop.

Readability Limits

Nested comprehensions with more than 2 levels become hard to read. Consider using explicit loops for complex logic.

🔑 Dict Comprehensions

Dictionary comprehensions use the same concise syntax to build dictionaries. You can create, transform, filter, or invert dictionaries elegantly.

dict_comp_examples.py
# Create a dict from a list
words = ['apple', 'banana', 'cherry']
word_lengths = {w: len(w) for w in words}
# Result: {'apple': 5, 'banana': 6, 'cherry': 6}

# Invert key-value pairs
original = {'a': 1, 'b': 2, 'c': 3}
inverted = {v: k for k, v in original.items()}
# Result: {1: 'a', 2: 'b', 3: 'c'}

# Filter dict by values
scores = {'Alice': 85, 'Bob': 92, 'Charlie': 78}
high_scores = {k: v for k, v in scores.items() if v >= 80}
# Result: {'Alice': 85, 'Bob': 92}

# Transform dict values
temps_c = {'NYC': 20, 'LA': 25, 'Boston': 15}
temps_f = {city: c * 9/5 + 32 for city, c in temps_c.items()}
# Result: {'NYC': 68.0, 'LA': 77.0, 'Boston': 59.0}

Dict comprehension syntax: {key: value for item in iterable}. You must provide both key and value expressions, separated by a colon.

Common Pattern

Inverting dictionaries with {v: k for k, v in dict.items()} is so useful that many codebases rely on it for lookup tables.

⚪ Set Comprehensions

Set comprehensions are like list comprehensions but produce sets instead of lists. They automatically deduplicate and unorder elements—perfect for uniqueness operations.

set_comp_examples.py
# Get unique squares
numbers = [1, 2, 2, 3, 3, 3]
unique_squares = {x ** 2 for x in numbers}
# Result: {1, 4, 9} - duplicates automatically removed

# Unique characters in strings
words = ['hello', 'world', 'python']
unique_chars = {c for word in words for c in word}
# Result: {'h', 'e', 'l', 'o', 'w', 'r', 'd', 'p', 'y', 't', 'n'}

# Filter for unique values above threshold
scores = [95, 85, 95, 75, 85, 90]
high_grades = {s for s in scores if s >= 85}
# Result: {85, 90, 95} - no duplicates, sorted doesn't apply to sets

# Intersection of unique values
list1 = [1, 2, 3, 3]
list2 = [2, 3, 4, 4]
common = {x for x in list1 if x in list2}
# Result: {2, 3}
Automatic Deduplication

Set comprehensions are perfect when you want unique values. The set automatically removes duplicates during creation, making them faster than filtering lists manually.

⚡ Generator Expressions

Generator expressions use parentheses instead of brackets. They don't create a list immediately—instead, they generate values on-demand, saving memory for large datasets.

generator_examples.py
# List comprehension - creates entire list in memory
squares_list = [x ** 2 for x in range(1000000)]

# Generator - computes values lazily
squares_gen = (x ** 2 for x in range(1000000))

# Consume generator values one at a time
for square in squares_gen:
    if square > 1000:
        print(square)
        break  # Stop early - rest are never computed!

# Generator in sum() - memory efficient
total = sum(x ** 2 for x in range(1000000))

# Convert generator to list when needed
gen = (x * 2 for x in [1, 2, 3])
result = list(gen)
# Result: [2, 4, 6]

Generator expressions use parentheses: (expr for item in iterable). Use them with functions like sum(), max(), or in loops where you don't need the entire list.

Memory Efficiency

Generators are perfect for processing large files or infinite sequences. They compute values on-the-fly, using constant memory regardless of dataset size.

❓ Conditional Expressions

Use ternary operators inside comprehensions to transform values conditionally. This produces different results based on conditions without filtering.

conditional_examples.py
# Transform with condition: positive=1, else 0
numbers = [-1, 0, 1, 2, -3]
signs = [1 if x > 0 else 0 for x in numbers]
# Result: [0, 0, 1, 1, 0]

# Label items by value
scores = [45, 78, 92, 55]
grades = ['Pass' if s = 60 else 'Fail' for s in scores]
# Result: ['Fail', 'Pass', 'Pass', 'Fail']

# Multiple conditions with elif
temps = [10, 20, 30, 40]
descriptions = [
    'Cold' if t < 15
    else 'Cool' if t < 25
    else 'Warm' if t < 35
    else 'Hot'
    for t in temps
]
# Result: ['Cold', 'Cool', 'Warm', 'Hot']

# Conditional transformation vs filtering
values = [1, 2, 3, 4]
doubled_or_zero = [x * 2 if x % 2 == 0 else 0 for x in values]
# Result: [0, 4, 0, 8] - same length, different values
if/else vs if filter

Use if/else when you want to transform (keep all items but change some). Use if without else to filter (reduce list length).

📌 Walrus Operator (:=)

Python 3.8+ introduced the walrus operator := for assignment expressions. Inside comprehensions, it lets you assign and use values in a single expression, reducing duplication.

walrus_examples.py
# Avoid computing the same expression twice
words = ['hello', 'world', 'python']

# Without walrus - compute len() twice
long_words_verbose = [(w, len(w)) for w in words if len(w) > 4]

# With walrus - compute len() once
long_words = [(w, l) for w in words if (l := len(w)) > 4]
# Result: [('hello', 5), ('world', 5), ('python', 6)]

# Avoid repeated expensive function calls
data = ['  hello  ', 'world', '  python  ']
cleaned = [s for item in data if (s := item.strip()).lower()] for x in [1]]
# Result: [stripped, lowercased strings]

# Use in dict comprehension
numbers = [1, 2, 3, 4, 5]
squares_dict = {x: sq for x in numbers if (sq := x ** 2) > 10}
# Result: {4: 16, 5: 25}

The walrus operator := assigns a value and returns it, all in one expression. This is most useful when the same computation appears multiple times in a comprehension.

Readability Trade-off

While walrus operators reduce duplication, they can make code less clear. Use them when the computation is expensive or repeated multiple times.

📦 Flattening Nested Structures

Comprehensions excel at flattening nested lists, dictionaries, and complex structures into simple, flat collections. This is a common data processing need.

flatten_examples.py
# Flatten 2D list
matrix = [[1, 2, 3], [4, 5], [6, 7, 8]]
flat = [x for row in matrix for x in row]
# Result: [1, 2, 3, 4, 5, 6, 7, 8]

# Flatten 3D list
cube = [
    [[1, 2], [3, 4]],
    [[5, 6], [7, 8]]
]
flat_cube = [
    x for plane in cube
    for row in plane
    for x in row
]
# Result: [1, 2, 3, 4, 5, 6, 7, 8]

# Flatten dict values (lists)
groups = {
    'A': [1, 2],
    'B': [3, 4],
    'C': [5]
}
all_values = [v for values in groups.values() for v in values]
# Result: [1, 2, 3, 4, 5]

# Flatten with filtering
nested = [[1, 2, 3], [4, 5], [6, 7]]
even_flat = [x for sublist in nested for x in sublist if x % 2 == 0]
# Result: [2, 4, 6]
Order Matters

In nested flattening, the order of for clauses determines the flattening direction. Read them left-to-right like nested loops.

⚠️ When NOT to Use Comprehensions

While comprehensions are powerful, they're not always the best choice. Readability matters more than cleverness. Know when to use explicit loops instead.

antipatterns.py
# AVOID: Too complex to read
result = [
    f(x) if g(x) else h(x) for x in data
    if p(x) and q(x) or r(x)
]

# BETTER: Use explicit loop
result = []
for x in data:
    if (p(x) and q(x)) or r(x):
        if g(x):
            result.append(f(x))
        else:
            result.append(h(x))

# AVOID: Side effects in comprehension
squared = [print(x) or x**2 for x in numbers]  # Ugly!

# BETTER: Separate concerns
for x in numbers:
    print(x)
squared = [x**2 for x in numbers]

# AVOID: Deeply nested (3+ levels)
bad = [
    x for a in l1 for b in l2 for c in l3
    if check(a, b, c)
    for x in process(a, b, c)
]

# BETTER: Use explicit loops for clarity
good = []
for a in l1:
    for b in l2:
        for c in l3:
            if check(a, b, c):
                good.extend(process(a, b, c))
Readability First

If a comprehension is hard to understand, use a loop instead. Code is read far more often than it's written. Prioritize clarity over cleverness.

🌍 Real-World Patterns

Comprehensions shine in real-world data processing. Here are patterns you'll use constantly: transforming API responses, batch operations, and data pipelines.

realworld_patterns.py
# Pattern 1: Extract from API response
api_response = [
    {'id': 1, 'name': 'Alice', 'email': 'alice@example.com'},
    {'id': 2, 'name': 'Bob', 'email': 'bob@example.com'}
]
emails = [user['email'] for user in api_response]

# Pattern 2: Filter by condition
products = [
    {'name': 'Laptop', 'price': 1200},
    {'name': 'Mouse', 'price': 25}
]
expensive = [p for p in products if p['price'] > 100]

# Pattern 3: Batch operation with retry
def process_safely(item):
    try:
        return process(item)
    except:
        return None

items = ['item1', 'item2', 'item3']
results = [process_safely(item) for item in items if process_safely(item) is not None]

# Pattern 4: Data validation and cleaning
raw_data = ['Alice', '', 'Bob', '   ', 'Charlie']
cleaned = [name.strip() for name in raw_data if name.strip()]

# Pattern 5: Create lookup table
users = [{'id': 1, 'name': 'Alice'}, {'id': 2, 'name': 'Bob'}]
id_to_name = {u['id']: u['name'] for u in users}

# Pattern 6: Group and deduplicate
tags = ['python', 'javascript', 'python', 'rust', 'javascript']
unique_tags = {t for t in tags}

These patterns appear in every Python project. Mastering them will make your data processing code clean, fast, and Pythonic.

Pro Tip

Combine comprehensions with built-in functions like sorted(), map(), and filter() for composable pipelines. Comprehensions are often clearer than function chaining.
