🏗️ Creating Dicts

Dictionaries are one of Python's most powerful data structures. They store data as key-value pairs, allowing you to access values by their unique keys instead of by position. Unlike lists that use integer indices, dictionaries give you semantic meaning through custom keys.

create_dicts.py
# Literal syntax with curly braces
person = {
    'name': 'Alice',
    'age': 30,
    'city': 'Portland'
}

# Using dict() constructor
settings = dict(theme='dark', language='en')

# Empty dictionary
empty = {}

# From list of tuples
pairs = [('x', 10), ('y', 20)]
coords = dict(pairs)

# Using dict.fromkeys() with default value
template = dict.fromkeys(['a', 'b', 'c'], 0)
Key Facts

Dictionary keys must be immutable (strings, numbers, tuples). Values can be any Python object. Keys are unique—duplicate keys overwrite the previous value. As of Python 3.7+, dictionaries maintain insertion order.

🔍 Accessing Values

There are multiple ways to retrieve values from a dictionary, each with different behaviors when a key doesn't exist.

access_values.py
user = {'name': 'Bob', 'email': 'bob@example.com'}

# Direct access with bracket notation
name = user['name']  # 'Bob'

# Accessing non-existent key raises KeyError
# name = user['phone']  # ERROR: KeyError

# Using .get() returns None if key doesn't exist
phone = user.get('phone')  # None

# .get() with a default value
phone = user.get('phone', 'Not provided')

# .setdefault() returns value or sets it if missing
user.setdefault('age', 25)  # Returns 25 and adds to dict
Method Behavior Returns if Missing
dict[key] Direct access KeyError (crashes)
.get(key) Safe access None
.get(key, default) Safe with fallback Your default value
.setdefault(key, value) Get + set if missing Adds key to dict

✏️ Adding & Updating

Dictionaries are mutable, so you can add new key-value pairs, modify existing values, or merge multiple dictionaries.

add_update.py
config = {'host': 'localhost', 'port': 8000}

# Add or update a single key-value pair
config['debug'] = True
config['port'] = 3000  # Updates existing key

# Merge another dict into this one with .update()
new_settings = {'timeout': 30, 'ssl': True}
config.update(new_settings)

# Python 3.9+: Use merge operator |
merged = config | new_settings

# Python 3.9+: In-place merge with |=
config |= new_settings
Updating vs. Merging

Use dict[key] = value to add/change a single item. Use .update() to add/change multiple items from another dict. The merge operator | creates a new dict without modifying the original.

🗑️ Removing Items

There are several ways to remove keys from a dictionary, each with different behavior when the key doesn't exist.

remove_items.py
data = {'a': 1, 'b': 2, 'c': 3}

# del removes a key; raises KeyError if not found
del data['a']

# .pop(key) removes and returns value; raises KeyError if not found
value = data.pop('b')  # Returns 2

# .pop(key, default) removes key or returns default
value = data.pop('z', 'Not found')

# .popitem() removes and returns last added key-value pair
key, val = data.popitem()

# .clear() removes all items
data.clear()
Method Returns If Key Missing
del dict[key] Nothing (in-place) KeyError
.pop(key) The value KeyError
.pop(key, default) Value or default Returns default
.popitem() Last (key, value) KeyError if empty
.clear() Nothing N/A

🔄 Iteration

Dictionaries are iterable. You can loop through keys, values, or both using different methods.

dict_iteration.py
scores = {'Alice': 95, 'Bob': 87, 'Carol': 92}

# Iterate over keys (default)
for name in scores:
    print(name)  # Alice, Bob, Carol

# Iterate over values using .values()
for score in scores.values():
    print(score)  # 95, 87, 92

# Iterate over key-value pairs using .items()
for name, score in scores.items():
    print(f'{name}: {score}')

# Check if key exists
if 'Alice' in scores:
    print(scores['Alice'])
Three Ways to Loop

for key in dict: gives keys • for value in dict.values(): gives values • for key, value in dict.items(): gives both. Use in to check key existence.

🎯 Dict Comprehensions

Dict comprehensions let you create dictionaries dynamically, transforming data or filtering based on conditions in a single elegant expression.

dict_comprehensions.py
# Basic: map values from a list
numbers = [1, 2, 3, 4]
squares = {n: n**2 for n in numbers}
# {1: 1, 2: 4, 3: 9, 4: 16}

# With condition: only even numbers
evens = {n: n*10 for n in numbers if n % 2 == 0}

# Swap keys and values
original = {'a': 1, 'b': 2}
swapped = {v: k for k, v in original.items()}

# From two lists (keys and values)
names = ['Alice', 'Bob']
ages = [30, 25]
people = {name: age for name, age in zip(names, ages)}

🪆 Nested Dicts

Dictionaries can contain other dictionaries as values, creating nested structures perfect for representing complex data with multiple levels of organization.

nested_dicts.py
# Nested dictionary structure
company = {
    'name': 'TechCorp',
    'employees': {
        'alice': {'role': 'Engineer', 'salary': 100000},
        'bob': {'role': 'Manager', 'salary': 120000}
    }
}

# Access nested values
alice_role = company['employees']['alice']['role']

# Update nested value
company['employees']['alice']['salary'] = 110000

# Iterate through nested structure
for emp_id, emp_data in company['employees'].items():
    print(f'{emp_id}: {emp_data['role']}')
KeyError with Nested Access

When accessing nested dicts with bracket notation, each level must exist. Use .get() for safer nested access: company.get('employees', {}).get('alice', {}).get('role')

🔮 Common Patterns

Dictionaries excel at solving real-world problems. Here are practical patterns you'll use frequently.

dict_patterns.py
# Pattern 1: Counting occurrences
text = 'hello world'
char_count = {}
for char in text:
    char_count[char] = char_count.get(char, 0) + 1

# Pattern 2: Grouping by category
fruits = [{'name': 'apple', 'color': 'red'},
            {'name': 'banana', 'color': 'yellow'}]
by_color = {}
for fruit in fruits:
    color = fruit['color']
    by_color.setdefault(color, []).append(fruit['name'])

# Pattern 3: Inverting a dict
mapping = {'a': 1, 'b': 2}
inverted = {v: k for k, v in mapping.items()}

# Pattern 4: Merging multiple dicts
dict1 = {'a': 1}
dict2 = {'b': 2}
merged = {**dict1, **dict2}  # {'a': 1, 'b': 2}

📊 Sorting Dicts

While dicts maintain insertion order, you often need to sort them by keys or values. The sorted() function is your tool.

sort_dicts.py
scores = {'Alice': 95, 'Carol': 92, 'Bob': 87}

# Sort by keys (alphabetically)
sorted_keys = sorted(scores.keys())
# ['Alice', 'Bob', 'Carol']

# Sort by values (ascending)
sorted_by_value = sorted(scores, key=scores.get)

# Sort by values (descending)
sorted_desc = sorted(scores, key=scores.get, reverse=True)

# Create new sorted dict from items
sorted_dict = dict(sorted(scores.items(), key=lambda item: item[1]))

# Complex sort with lambda
data = {'z': 5, 'a': 3, 'm': 8}
result = dict(sorted(data.items()))

🎲 DefaultDict & Counter

The collections module provides specialized dict subclasses for common patterns like handling missing keys and counting occurrences.

defaultdict_counter.py
from collections import defaultdict, Counter

# defaultdict: auto-create missing keys with default value
groups = defaultdict(list)
groups['python'].append('Alice')  # No KeyError!

# defaultdict with int
counts = defaultdict(int)
for word in ['cat', 'dog', 'cat']:
    counts[word] += 1

# Counter: specialized for counting hashable objects
letters = Counter('banana')
# Counter({'a': 3, 'n': 2, 'b': 1})

# Counter methods
top_3 = letters.most_common(3)
total = sum(letters.values())
When to Use Each

defaultdict is great for grouping or accumulating values. Counter is perfect for counting occurrences and finding most common items. Both save you from manual existence checking.

📚 Methods Reference

A quick reference of all the important dictionary methods and their signatures.

Method Purpose Example
.keys() Get all keys dict.keys() → dict_keys
.values() Get all values dict.values() → dict_values
.items() Get all key-value pairs dict.items() → dict_items
.get(key, default) Safe access with default dict.get('key', 0)
.setdefault(key, value) Get or set if missing dict.setdefault('key', 10)
.pop(key, default) Remove and return dict.pop('key', None)
.popitem() Remove last item dict.popitem() → (key, value)
.update(other) Merge another dict dict.update({'x': 1})
.clear() Remove all items dict.clear()
.copy() Shallow copy new_dict = dict.copy()

🌍 Real-World Patterns

Dictionaries power real applications. Here are practical patterns you'll encounter in production code.

real_world_patterns.py
# Pattern 1: Caching/Memoization
cache = {}
def expensive_function(n):
    if n in cache:
        return cache[n]
    result = n ** 2
    cache[n] = result
    return result

# Pattern 2: Configuration objects
app_config = {
    'database': {'host': 'localhost', 'port': 5432},
    'logging': {'level': 'INFO'},
    'features': {'auth': True, 'api': True}
}

# Pattern 3: Data grouping for display
students = [
    {'name': 'Alice', 'grade': 'A'},
    {'name': 'Bob', 'grade': 'B'}
]
by_grade = defaultdict(list)
for student in students:
    by_grade[student['grade']].append(student['name'])

# Pattern 4: API response handling
response = {'status': 'success', 'data': {...}}
status = response.get('status', 'unknown')
data = response.get('data', {})
