🏗️ Classes & Objects

A class is a blueprint for creating objects. It defines the structure (attributes) and behavior (methods) that objects will have. Objects are instances of classes—individual examples created from the blueprint.

Creating a Simple Class

dog.py 
class Dog:
    """A class representing a dog."""

    def __init__(self, name, age):
        self.name = name
        self.age = age

    def bark(self):
        return f"{self.name} says: Woof!"

# Create instances (objects)
dog1 = Dog("Buddy", 3)
dog2 = Dog("Max", 5)

print(dog1.name)  # Output: Buddy
print(dog1.bark())  # Output: Buddy says: Woof!
Key Concepts
__init__: The constructor method, called automatically when creating an object. It initializes instance attributes.
self: Refers to the current instance. Always the first parameter in methods.
Instance attributes: Variables like self.name that store data specific to each object.

Understanding Object Creation

person.py 
class Person:
    def __init__(self, name, email):
        self.name = name
        self.email = email
        self.created_at = time.time()

# Each instance has its own data
alice = Person("Alice", "alice@example.com")
bob = Person("Bob", "bob@example.com")

# Different objects, different attributes
print(alice.name)  # Alice
print(bob.name)    # Bob
print(alice == bob)  # False (different objects)

⚙️ Instance Methods

Instance methods are functions defined within a class that operate on instance data. They have access to self and can read/modify instance attributes.

Defining and Calling Methods

bank_account.py 
class BankAccount:
    def __init__(self, owner, balance=0):
        self.owner = owner
        self.balance = balance

    def deposit(self, amount):
        self.balance += amount
        return f"Deposited ${amount}. New balance: ${self.balance}"

    def withdraw(self, amount):
        if amount > self.balance:
            return "Insufficient funds"
        self.balance -= amount
        return f"Withdrew ${amount}. New balance: ${self.balance}"

account = BankAccount("John", 1000)
print(account.deposit(500))   # Deposited $500. New balance: $1500
print(account.withdraw(200))  # Withdrew $200. New balance: $1300

Method Chaining

string_builder.py 
class StringBuilder:
    def __init__(self):
        self.text = ""

    def append(self, text):
        self.text += text
        return self  # Return self for chaining

    def uppercase(self):
        self.text = self.text.upper()
        return self

    def get(self):
        return self.text

# Chain methods together
builder = StringBuilder()
result = builder.append("Hello").append(" ").append("World").uppercase().get()
print(result)  # HELLO WORLD

🔄 Class vs Instance Variables

Instance variables are unique to each object. Class variables are shared across all instances. Understanding the difference is crucial for managing state correctly.

Comparing Variable Types

counter.py 
class Counter:
    total_created = 0  # Class variable

    def __init__(self, name):
        self.name = name  # Instance variable
        self.count = 0   # Instance variable
        Counter.total_created += 1

c1 = Counter("First")
c2 = Counter("Second")
c3 = Counter("Third")

# Each instance has different instance variables
print(c1.name)  # First
print(c2.name)  # Second

# But all share the same class variable
print(Counter.total_created)  # 3
print(c1.total_created)       # 3 (accessed via instance)

Mutable Class Variables (Caution!)

team.py 
class Team:
    members = []  # DANGEROUS: mutable class variable

    def __init__(self, name):
        self.name = name

    def add_member(self, member):
        Team.members.append(member)

team1 = Team("A")
team2 = Team("B")

team1.add_member("Alice")
team2.add_member("Bob")

# Oops! Both teams share the same list
print(Team.members)  # ['Alice', 'Bob']

# Solution: Use instance variables
class Team:
    def __init__(self, name):
        self.name = name
        self.members = []  # Each instance has its own list

🔗 Inheritance

Inheritance allows a class to inherit attributes and methods from another class. The parent (base) class provides common functionality, and child (derived) classes specialize it.

Basic Inheritance

animals.py 
class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        return f"{self.name} makes a sound"

class Dog(Animal):  # Dog inherits from Animal
    def speak(self):  # Override parent method
        return f"{self.name} barks"

class Cat(Animal):
    def speak(self):
        return f"{self.name} meows"

dog = Dog("Buddy")
cat = Cat("Whiskers")

print(dog.speak())  # Buddy barks
print(cat.speak())  # Whiskers meows

Using super()

vehicle.py 
class Vehicle:
    def __init__(self, brand, color):
        self.brand = brand
        self.color = color

class Car(Vehicle):
    def __init__(self, brand, color, num_doors):
        super().__init__(brand, color)  # Call parent __init__
        self.num_doors = num_doors

    def info(self):
        return f"{self.brand} {self.color} car with {self.num_doors} doors"

car = Car("Toyota", "Red", 4)
print(car.info())  # Toyota Red car with 4 doors

🔀 Multiple Inheritance

A class can inherit from multiple parent classes. Python uses Method Resolution Order (MRO) to determine which method to call when multiple parents define the same method.

Multiple Inheritance & MRO

mro_example.py 
class Flyer:
    def move(self):
        return "Flying"

class Swimmer:
    def move(self):
        return "Swimming"

class Duck(Flyer, Swimmer):  # Flyer first, then Swimmer
    pass

duck = Duck()
print(duck.move())  # Flying (Flyer is first in MRO)

# Check MRO
print(Duck.__mro__)
# (, , , )

Mixins Pattern

mixins.py 
class JSONMixin:
    def to_json(self):
        import json
        return json.dumps(self.__dict__)

class TimestampMixin:
    def get_age(self):
        import time
        return time.time() - self.created_at

class User(JSONMixin, TimestampMixin):
    def __init__(self, name):
        self.name = name
        self.created_at = __import__("time").time()

user = User("Alice")
print(user.to_json())  # Uses JSONMixin
print(user.get_age())    # Uses TimestampMixin

✨ Properties & Decorators

The @property decorator allows you to define methods that can be accessed like attributes. This enables validation and computed attributes while maintaining clean syntax.

@property Decorator

temperature.py 
class Temperature:
    def __init__(self, celsius):
        self._celsius = celsius

    @property
    def fahrenheit(self):
        return self._celsius * 9/5 + 32

    @fahrenheit.setter
    def fahrenheit(self, value):
        self._celsius = (value - 32) * 5/9

temp = Temperature(0)
print(temp.fahrenheit)      # 32.0 (accessed like attribute)
temp.fahrenheit = 212        # Set via setter
print(temp._celsius)         # 100.0

Validation with Properties

age.py 
class Person:
    def __init__(self, age):
        self.age = age  # Uses setter

    @property
    def age(self):
        return self._age

    @age.setter
    def age(self, value):
        if value < 0 or value > 150:
            raise ValueError("Invalid age")
        self._age = value

person = Person(25)
person.age = 200  # Raises ValueError

📌 Class & Static Methods

@classmethod operates on the class itself (receives the class as the first argument), while @staticmethod doesn't receive the class or instance. Use them to organize utility functions and alternative constructors.

@classmethod Example

date_factory.py 
class Date:
    instances_created = 0

    def __init__(self, day, month, year):
        self.day = day
        self.month = month
        self.year = year
        Date.instances_created += 1

    @classmethod
    def from_string(cls, date_str):
        """Factory method: create from '12-25-2023' format"""
        day, month, year = date_str.split("-")
        return cls(int(day), int(month), int(year))

d1 = Date(25, 12, 2023)
d2 = Date.from_string("25-12-2023")
print(Date.instances_created)  # 2

@staticmethod Example

math_utils.py 
class MathUtils:
    @staticmethod
    def is_prime(n):
        """Check if number is prime"""
        if n < 2:
            return False
        for i in range(2, int(n**0.5) + 1):
            if n % i == 0:
                return False
        return True

    @staticmethod
    def gcd(a, b):
        """Greatest common divisor"""
        while b:
            a, b = b, a % b
        return a

# Call without instance
print(MathUtils.is_prime(17))   # True
print(MathUtils.gcd(48, 18))  # 6

🔮 Dunder Methods

Dunder (double underscore) methods like __str__ and __add__ enable your objects to behave like built-in types. They allow custom string representation, comparison, arithmetic, and more.

String Representation

vector.py 
class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __str__(self):
        """User-friendly representation"""
        return f"Vector({self.x}, {self.y})"

    def __repr__(self):
        """Developer-friendly, evaluable representation"""
        return f"Vector({self.x!r}, {self.y!r})"

    def __len__(self):
        return int((self.x**2 + self.y**2)**0.5)

v = Vector(3, 4)
print(v)         # Vector(3, 4) (__str__)
print(repr(v))  # Vector(3, 4) (__repr__)
print(len(v))   # 5 (__len__)

Comparison & Arithmetic

fraction.py 
class Fraction:
    def __init__(self, num, den):
        self.num = num
        self.den = den

    def __eq__(self, other):
        return self.num * other.den == self.den * other.num

    def __lt__(self, other):
        return self.num * other.den < self.den * other.num

    def __add__(self, other):
        num = self.num * other.den + self.den * other.num
        den = self.den * other.den
        return Fraction(num, den)

f1 = Fraction(1, 2)
f2 = Fraction(1, 3)
print(f1 == Fraction(2, 4))  # True
print(f1 < f2)              # False

🔒 Encapsulation

Encapsulation is bundling data and methods together, and controlling access to them. Python uses naming conventions (_private, __mangled) to indicate access levels, as there's no true private keyword.

Private and Protected Attributes

bank.py 
class BankAccount:
    def __init__(self, owner, balance):
        self.owner = owner
        self._balance = balance  # Protected (by convention)
        self.__pin = "1234"     # Name-mangled (private)

    def withdraw(self, amount):
        """External method controls access to _balance"""
        if amount > self._balance:
            raise ValueError("Insufficient funds")
        self._balance -= amount

    def verify_pin(self, pin):
        """Only way to access __pin"""
        return pin == self.__pin

account = BankAccount("John", 1000)
# account._balance = 2000  # Discouraged (protected)
# account.__pin = "5678"  # Raises AttributeError (mangled)
print(account._BankAccount__pin)  # Name mangling: _ClassName__attr

Getters and Setters

student.py 
class Student:
    def __init__(self, name, gpa):
        self._name = name
        self._gpa = gpa

    def get_gpa(self):
        """Getter with validation"""
        return self._gpa

    def set_gpa(self, gpa):
        """Setter with validation"""
        if not (0 <= gpa <= 4.0):
            raise ValueError("GPA must be between 0 and 4.0")
        self._gpa = gpa

student = Student("Alice", 3.8)
print(student.get_gpa())  # 3.8

📐 Abstract Classes

Abstract classes define an interface that subclasses must implement. Use abc.ABC and @abstractmethod to enforce a contract, ensuring derived classes implement required methods.

Defining Abstract Classes

shapes.py 
from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def area(self):
        """All shapes must implement area()"""
        pass

    @abstractmethod
    def perimeter(self):
        """All shapes must implement perimeter()"""
        pass

    def describe(self):
        """Concrete method (optional in abstract class)"""
        return f"I am a {self.__class__.__name__}"

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius

    def area(self):
        return 3.14 * self.radius ** 2

    def perimeter(self):
        return 2 * 3.14 * self.radius

# shape = Shape()  # TypeError: Can't instantiate abstract class
circle = Circle(5)
print(circle.area())        # 78.5
print(circle.describe())  # I am a Circle

Abstract Properties

animal_abstract.py 
from abc import ABC, abstractmethod

class Animal(ABC):
    @property
    @abstractmethod
    def species(self):
        """Species must be defined"""
        pass

    @abstractmethod
    def make_sound(self):
        pass

class Dog(Animal):
    @property
    def species(self):
        return "Canis familiaris"

    def make_sound(self):
        return "Woof!"

dog = Dog()
print(dog.species)  # Canis familiaris

📦 Dataclasses

The @dataclass decorator automatically generates __init__, __repr__, __eq__, and other methods. It's perfect for simple data containers and reduces boilerplate code.

Basic Dataclass

point.py 
from dataclasses import dataclass

@dataclass
class Point:
    x: float
    y: float
    label: str = ""  # Default value

# Automatically generates __init__, __repr__, __eq__
p1 = Point(3, 4)
p2 = Point(3, 4)
p3 = Point(3, 4, "origin")

print(p1)          # Point(x=3, y=4, label='')
print(p1 == p2)    # True (auto-generated __eq__)
print(p3.label)   # origin

Frozen and Post-Init

config.py 
from dataclasses import dataclass, field

@dataclass(frozen=True)  # Immutable
class Config:
    host: str
    port: int
    tags: list = field(default_factory=list)

    def __post_init__(self):
        """Called after __init__, for validation/setup"""
        if self.port < 0 or self.port > 65535:
            raise ValueError("Invalid port")

cfg = Config("localhost", 8080)
# cfg.port = 9000  # Raises FrozenInstanceError
print(cfg)  # Config(host='localhost', port=8080, tags=[])

🎯 Design Patterns

Design patterns are reusable solutions to common OOP problems. Singleton ensures one instance, Observer enables event handling, Strategy allows swappable algorithms, and Composition avoids rigid inheritance.

Singleton Pattern

singleton.py 
class Logger:
    _instance = None

    def __new__(cls):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
        return cls._instance

    def log(self, msg):
        print(f"[LOG] {msg}")

log1 = Logger()
log2 = Logger()
print(log1 is log2)  # True (same instance)

Observer Pattern

observer.py 
class Subject:
    def __init__(self):
        self._observers = []

    def attach(self, observer):
        self._observers.append(observer)

    def notify(self, event):
        for observer in self._observers:
            observer.update(event)

class Observer:
    def update(self, event):
        raise NotImplementedError

class EmailObserver(Observer):
    def update(self, event):
        print(f"Email notification: {event}")

subject = Subject()
email_obs = EmailObserver()
subject.attach(email_obs)
subject.notify("User logged in")  # Email notification: User logged in

Composition Over Inheritance

composition.py 
class Engine:
    def start(self):
        return "Engine started"

class Transmission:
    def shift(self, gear):
        return f"Shifted to {gear}"

# Composition: Car HAS-A engine, not IS-A engine
class Car:
    def __init__(self):
        self.engine = Engine()
        self.transmission = Transmission()

    def start(self):
        return self.engine.start()

    def drive(self, gear):
        return self.transmission.shift(gear)

car = Car()
print(car.start())       # Engine started
print(car.drive("D"))  # Shifted to D
